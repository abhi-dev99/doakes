# Documentation: Frontend React Dashboard (`Appv2.jsx`)

## 1. Purpose
The frontend serves as the **Command Center** for fraud analysts. It is a real-time, React-based dashboard built with Vite and Tailwind CSS. It connects to the backend FastAPI server via WebSockets to stream live transactions and provides deep-dive analytics into fraud cases.

## 2. Architecture & Tech Stack
*   **Core**: React 18 with Vite build tool.
*   **Styling**: Tailwind CSS with custom CSS variables (in `index.css`) to enforce an Apple-inspired "glassmorphism" aesthetic.
*   **Data Visualization**: `recharts` for performance graphs, risk distributions, and volume trends.
*   **Icons**: `lucide-react` for consistent, crisp SVG iconography.
*   **Networking**: Native `fetch` API for REST calls and a custom `WebSocketClient` class (`api.js`) for persistent real-time streaming with automatic reconnection logic.

## 3. Key Components (`Appv2.jsx`)

### A. The Three Main Views
The dashboard is split into three core navigational views:
1.  **Dashboard (`DashboardView`)**: The high-level executive summary. Shows total volume, block rates, risk distributions, and recent critical alerts.
2.  **Live Traffic (`LiveTrafficView`)**: A real-time stream of transactions. Analysts can pause the stream, filter by risk level (e.g., only show CRITICAL), and click on any transaction to open the deep-dive inspector.
3.  **Settings (`SettingsView`)**: Control panel to adjust backend engine parameters (e.g., strictness thresholds for velocity, minimum amount for ML scoring) via the `/api/config` REST endpoints.

### B. Transaction Inspector (`TransactionInspector`)
A high-fidelity modal that pops up when an analyst clicks a specific transaction. It provides:
*   **Entity Context**: User ID and execution timestamp.
*   **Network & Origin**: IP Address, Device ID, and Geolocated City/State.
*   **Execution Trace**: A breakdown of the decision engine's flow (Pre-Auth Heuristics -> ML Ensemble -> XAI Enrichment).
*   **Explainable AI output**: The human-readable sentences generated by `explainable_ai.py` (e.g., "Transaction amount is 5x higher than normal").
*   **Raw Logs**: Analysts can instantly copy the raw JSON payload for debugging or export it as a `.json` file.

## 4. Real-time Implementation Details
The `api.js` wrapper maintains a WebSocket connection to `ws://localhost:8000/ws`. 
When the backend processes a transaction, it broadcasts the result. The React app listens for the `transaction` event and unshifts the new data into the `liveTransactions` state array, ensuring a sub-second "live feed" experience.

## 5. UI/UX Design Philosophy
The entire application was built using a strict CSS Grid layout to ensure perfect column alignment in tables. It uses the `Inter` font family and relies heavily on subtle borders, backdrop-blur (`backdrop-blur-xl`), and dark mode contrasts to achieve an enterprise-grade, premium feel, moving away from standard generic Bootstrap themes.
